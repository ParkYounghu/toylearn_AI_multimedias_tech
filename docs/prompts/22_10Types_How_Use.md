## 가상 문제 상황
🎬 시나리오 챌린지: 의견 조율을 위한 신속한 웹 프로토타입 개발
1. 가상 문제 상황
👤 페르소나: 멀티미디어 콘텐츠 기업 '네오스트림'의 프론트엔드 개발자 (3년 차)
🏙 배경:
귀사의 콘텐츠 운영팀으로부터 "사내 영상 리뷰 및 승인용 웹 대시보드" 제작을 긴급하게 요청받았습니다.
현재 기획서가 완성되지 않은 상태이며, 기획팀은 "직관적인 UI", 디자인팀은 "트렌디한 다크 모드", 운영팀은 "데이터 밀집형 테이블"을 각각 주장하며 의견이 좁혀지지 않고 있습니다.
💥 핵심 문제:
말로만 회의를 진행하다 보니 서로 상상하는 결과물이 달라 의사결정이 지연되고 있습니다.
오늘 오후 4시 회의 전까지, 실제로 작동하는 형태의 결과물이 없으면 프로젝트 착수가 무기한 연기될 위기입니다.
디자인 툴(Figma 등)로 그리기에는 시간이 부족하고, 버튼 클릭 등 인터랙션이 가능한 HTML/CSS 기반의 퍼블리싱 결과물을 보여줘야만 이해관계자들을 설득할 수 있습니다.
🎯 해결 과제:
요구사항(영상 플레이어 영역, 타임스탬프 댓글 기능, 승인/반려 버튼, 다크 모드 스타일)을 포함한 **단일 페이지 웹 프로토타입 코드(HTML, CSS, JS)**를 AI를 통해 즉시 생성하십시오.
이 코드는 회의 시간 내에 즉석에서 수정 요청을 반영할 수 있을 만큼 구조가 명확해야 합니다.
2. 훈련 초점 (힌트)
이 문제는 [코딩 및 프로토타입 생성 / 시각화] 능력을 평가
단순한 텍스트 생성이 아니라, 구체적인 UI/UX 요구사항을 구조화된 언어(코드)로 변환하는 프롬프트 엔지니어링 기술이 필요
## 문제 해결 프롬프트

너는 멀티미디어 콘텐츠 기업 **'네오스트림'의 프론트엔드 개발자** 역할을 수행한다. 오늘 오후 4시 회의에서 프로젝트 착수를 확정 짓기 위한 **최종 회의용 프로토타입**을 생성해야 한다.

**[상황 배경]**
기획/디자인/운영팀의 의견 충돌(직관성 vs 다크 모드 vs 데이터 밀집)로 프로젝트가 지연되고 있다. 개발자로서 이 상충되는 요구사항을 **단일 프로토타입**으로 통합하여 의사결정을 이끌어내야 한다.

**[핵심 목표]**
1.  **3팀 요구사항 통합 (조건부 생성):** '다크 모드' (디자인팀), '데이터 밀집형 목록' (운영팀), '직관적인 UI' (기획팀)를 모두 만족시킬 것.
2.  **즉시 작동 및 수정 가능 (제약 조건):** Tailwind CSS를 사용하여 코드가 간결하고, 회의 중 **즉석 수정**이 가능하도록 구조화할 것.
3.  **UX 주도적 개선:** 댓글의 타임스탬프 클릭 시 영상 위치로 바로 이동하는 기능(UX 핵심)을 구현할 것.

**1. 분석 및 전략 목표 (CoT, 인스턴스 기반 필수):**
* **Chain of Thought (6):** '요구사항 충돌 분석' → '다크/라이트 모드 토글로 유연성 확보' → '타임스탬프 이동 기능 구현' → '최종 프로토타입 산출'의 단계를 밟을 것.
* **인터랙션 구현 지시 (추가):** 승인/반려 버튼, 댓글 등록, 영상 로딩 시 **`alert()` 대신 커스텀 알림 UI**를 통해 사용자에게 명확한 피드백을 제공할 것.
* **인스턴스 기반 (9):** 대시보드 구조 및 데이터 밀집형 테이블 디자인은 **유튜브 스튜디오 또는 넷플릭스 리뷰 대시보드**의 레이아웃(좌측 영상, 우측 데이터)을 참고할 것.

**2. 출력 형식 및 예시 기반 통제 (단일 HTML 파일):**
답변은 반드시 하나의 **HTML 코드 블록**을 사용하며, 모든 CSS(Tailwind)와 JavaScript 로직은 단일 파일 내부에 포함해야 한다.

* **포함 요소 (2):** 영상 플레이스홀더 영역, 타임스탬프 댓글 목록, 승인/반려 버튼, 영상 목록 테이블을 명확하게 분리하여 구현할 것.
* **추가 기능 (UX 보완):**
    * **테마 토글:** 헤더 영역에 다크/라이트 모드 전환이 가능한 **토글 버튼**을 구현할 것.
    * **타임스탬프 점프:** 댓글의 시간 태그를 클릭하면 **mock 시간**이 해당 시간으로 이동하도록 JS 함수를 구현할 것.

**3. 최종 점검 (질문-응답 추가):**
* 프로토타입 생성 완료 후, **'Q: 이 프로토타입이 상충하는 3팀의 요구사항(디자인/운영/기획)을 모두 해결하는가?'**에 대해 **'A: (평가 및 이유)'**를 작성하시오.
